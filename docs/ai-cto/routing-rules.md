# 极享OS AI CTO 路由规则

## 路由顺序

```text
任务类型 → 影响模块 → 风险等级 → 能力角色 → 当前推荐模型 → 审查角色
```

模型是能力角色的当前映射，不是永久绑定；任务单必须记录实际使用的模型。

## 任务类型

| 类型 | 判断信号 | 默认能力角色 | 默认审查 |
| --- | --- | --- | --- |
| 探索验证 | 原型、页面尝试、MVP、方案比较 | 快速执行者 | 快速回归 |
| 产品设计 | 新模块、业务流程、需求澄清 | 产品设计者 | 架构负责人按需参与 |
| 常规开发 | 表单、列表、普通接口、局部功能 | 实施工程师 | 独立审查者 |
| 核心业务 | CRM、订单、财务、权限、跨模块状态 | 架构负责人 + 实施工程师 | 独立审查者 |
| 缺陷修复 | Bug、异常、性能和回归故障 | 实施工程师 | 独立审查者 |
| 发布审计 | 上线、迁移、大版本和生产检查 | 独立审查者 | 架构负责人或发布负责人 |

## 风险升级表

| 影响范围 | 最低风险 | 强制要求 |
| --- | --- | --- |
| 文档、只读分析、无业务状态变化 | R0 | 保留来源和结论 |
| 低风险 UI 或局部可回退改动 | R1 | 人工批准任务单后实施 |
| 权限、数据范围、公共池、客户归属或线索分配 | R2 | 人工批准、权限回归、独立审查 |
| 正式订单、订单审核、订单入库或订单分账 | R2 | 产品/架构判断、状态回归、独立审查 |
| 售后挽回、提成、财务状态或分账 | R2 | 财务口径确认、数据权限检查、独立审查 |
| Prisma schema、迁移、跨模块状态流转 | R2 | 迁移兼容性、备份/回滚判断、独立审查 |
| 生产部署、生产数据、环境变量或恢复操作 | R3 | 关键动作前二次人工确认、发布门禁、独立审查 |
| 数据删除、破坏性迁移、资金、正式公开发布 | R3 | 二次确认、明确回滚/停止条件和独立审查 |

## 能力角色与当前模型

| 能力角色 | 当前推荐模型 | 极享OS中的用途 |
| --- | --- | --- |
| 快速执行者 | GPT-5.5 | 低风险 UI、原型、局部代码调整 |
| 产品设计者 | GPT-5.6 Luna | 业务流程、需求澄清、验收标准 |
| 架构负责人 | GPT-5.6 Sol | 权限、订单、财务、数据库和跨模块设计 |
| 实施工程师 | GPT-5.5 或 GPT-5.6 Sol | 按风险和技术范围编码、测试 |
| 独立审查者 | GPT-5.6 Terra | 代码、安全、回归和发布审查 |

以上模型名称是当前候选映射，不代表当前账户一定可用。实际模型不可用时，按相同能力角色选择替代，并记录原模型、替代模型和原因。

## 模型不可用时的替代规则

1. 不改变任务风险等级和审批要求。
2. 不用低风险快速执行者替代核心业务的架构负责人或独立审查者。
3. 在任务单的“替代模型记录”字段写明替代事实。
4. 如果找不到等价能力，任务进入“已阻塞”，等待用户选择或提供模型。

## 独立审查规则

- 开发执行者不得充当同一阶段的最终独立审查者。
- R2 任务至少审查业务边界、权限/数据范围、状态流转、回归和测试证据。
- R3 任务还要审查生产影响、备份、停止条件、回滚和人工确认记录。
- 审查结果必须回写任务文件，不能只留在聊天中。

## 重新规划触发器

- 用户需求增加模块、字段、角色、状态或外部动作。
- 风险从 R0/R1 升到 R2/R3。
- 数据库、权限、订单、财务或生产方案发生实质变化。
- 同一实现方案连续测试失败两次。
- 发现事实源冲突，且无法由现有文档判断当前口径。
- 现有验收标准无法覆盖实际结果。
