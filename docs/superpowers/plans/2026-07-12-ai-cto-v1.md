# AI CTO V1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 建立一个先服务极享OS、以后可复用到其他项目的 Markdown AI CTO 调度闭环，使自然语言需求先生成待审批任务单，人工批准后才允许实施。

**Architecture:** 通用治理规则保存在 `AI-Project-Growth` 仓库，极享OS仓库只保存项目上下文、模型路由、启动入口和任务记录。AI CTO 先按能力角色完成路由，再读取可替换的当前模型映射；任务状态、审批和执行证据全部落在 Markdown 任务文件中。

**Tech Stack:** Markdown、Git、POSIX shell 检查命令；极享OS项目上下文为 React、Express、Prisma、MySQL。

## Global Constraints

- AI CTO 默认不直接开发，必须先完成分类、风险判断、任务编排和验收定义。
- 第一版只使用 Markdown，不修改极享OS应用代码，不新增数据库表，不调用模型 API。
- 通用治理规范与极享OS项目配置分开存储。
- 模型路由使用“能力角色 + 当前推荐模型”双层结构。
- 待审批任务未经人工明确批准不得进入实施。
- R2 任务需要人工批准和独立审查；R3 任务还需在关键执行动作前再次确认。
- 权限、财务、订单或生产数据任务最低为 R2；数据删除、迁移、资金、公开发布或生产部署为 R3。
- 模型不可用时按能力角色选择替代模型，并在任务单中记录替代原因。
- 聊天记录不是最终事实源，任务文件才是。
- 保留两个仓库中与本任务无关的已有修改和未跟踪文件。
