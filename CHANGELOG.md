# Changelog

## Unreleased - 2026-07-16

### 业务附件与售后挽回

- 订单付款截图限制为 1 张，订单聊天/成交凭证支持多选和粘贴截图，最多 8 张。
- 售后挽回收款凭证、聊天记录分别支持多选和粘贴截图，最多 8 张；客户姓名必填，手机号或微信至少填写一项。
- 系统设置新增售后来源平台与店铺配置，挽回订单按平台联动选择店铺，停用项不影响历史单据展示。
- 交付步骤附件支持真实上传、鉴权查看、下载、删除和重新上传；步骤内容增加独立保存、未保存提示和关闭提醒。
- 交付列表新增实际完成时间，已完成步骤展示完成时间和完成人。
- 业务附件存放在非公开目录，业务记录与列表仅保留公开元数据，不保存 Base64、文件内容或内部存储路径。

## Unreleased - 2026-07-12

### 已验证改进

- 收紧系统设置、客户、订单、资产和 legacy storage 的服务端权限。
- 客户放入公海、领取、分配以及线索核心写操作迁移到记录级服务端命令。
- 订单申请、审核、提成生成和交付创建增加事务与幂等处理。
- 财务只读页不再因隐式写入持续出现 Forbidden / invalid storage key 告警。
- 发布脚本增加生产配置、备份、迁移和持久化目录安全门禁。
- 修复资产命令服务的可选使用人类型，恢复 TypeScript 和生产构建。
- 客户创建增加姓名、手机号和长度校验，并用规范化手机号的确定性指纹防止并发重复创建。
- 客户冲突响应现在保留 HTTP 409 状态。
- 订单负责人候选改由服务端按本人/部门/全部数据范围返回。
- 订单分账与售后挽回分账统一读取可分配人员、部门和职位，选择人员后即时显示部门。
- 财务分账在判断源订单前主动同步服务端订单，消除首次进入的误删除提示。
- 客户新增支持手机号或微信二选一；保存失败时保留表单并显示服务端原因。
- 新增独立“领取公海客户”权限，销售领取不再依赖客户分配权限。
- 公海池页面隐藏“新增客户”按钮，正常客户列表保持不变。
- 新增受控客户标签目录：超级管理员维护分组、单选/多选规则、线索/客户适用范围、启停、改名、排序和合并。
- 线索与客户改为选择预设标签，服务端校验数据范围与目录状态；线索转客户仅继承适用于客户的共享标签。
- 客户列表新增 grouped/any/all、无标签和缺少指定标签组筛选，筛选参数支持刷新、分页与浏览器导航恢复。
- 提供历史自由文本标签迁移预览/校验和/apply 流程和回环地址限定的客户标签 QA 脚本。
- 目录规则或标签分组变更前会在共享事务锁内校验已有客户与线索分配，冲突时原子拒绝。
- 历史迁移预览新增结构化分配冲突，并与歧义名称一同阻止 apply。
- 新增安全的标签分组合并；组间同名标签必须由管理员先显式合并。
- 分组合并只允许启用的目标分组，但允许将历史停用源组治理到启用目标。
- 历史标签迁移覆盖 canonical LeadRecord 与 legacy BusinessRecord 线索，同 ID 优先 canonical 并防止双计/双写。
