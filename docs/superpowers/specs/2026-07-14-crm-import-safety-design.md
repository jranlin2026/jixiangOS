# CRM 导入员工与标签安全匹配设计

**日期：** 2026-07-14
**状态：** 已确认
**目标：** 用最少的改动保证 CRM 客户导入时负责人和预设标签都能唯一、稳定地绑定系统 ID，同时移除已失效入口。

## 背景

当前 CRM 导入允许负责人姓名在员工目录中不存在。系统会保留 Excel 中的姓名用于展示，但不会生成负责人 ID，因此后来创建同名员工也不会自动获得这些客户。当前标签预检能够按名称识别预设标签，但正式导入只保存标签名称，没有写入标签 ID，因此仍可能显示为“历史未归类”。

本次不改变“姓名只用于展示、ID 才决定归属”的基础规则，不增加自动认领或模糊匹配。

## 范围

### 1. 负责人导入强校验

- 只校验最终决定团队客户归属的负责人姓名，不因历史创建人、录入人、分配人等操作记录缺失而阻止导入。
- 公海客户不要求负责人匹配，统一保持公海归属。
- 团队客户的负责人姓名必须在当前在职、启用员工中唯一匹配。
- 找不到员工时禁止导入，并展示缺失负责人名单，引导用户先到组织架构创建员工后重新预检。
- 同一姓名匹配到多个在职员工时禁止导入，并展示重名负责人名单。因为原始 Excel 只有姓名，系统不得猜测员工 ID。
- 校验同时存在于前端导入流程和服务端写入入口，避免绕过页面直接提交未匹配数据。
- 校验通过后，导入客户直接写入 `ownerId`、负责人姓名快照和 `ownerIdentityStatus: resolved`。

### 2. 不增加未匹配客户展示

- 新导入被强校验拦截，正常流程不会再生成负责人未匹配客户。
- 普通新增、领取和分配客户继续使用现有员工 ID 逻辑。
- 保留底层未匹配状态作为历史数据和异常数据的安全兜底，但不增加客户列表徽标、筛选器或新的批量处理页面。
- 当前测试数据由用户在新版完成后清空并重新导入，不在本次代码中自动修改或认领。

### 3. 标签精确绑定

- 读取 Excel 的“标签”和“客户标签”列。
- 多标签继续按逗号、中文逗号、顿号、分号、斜杠和竖线拆分，并清理首尾空格。
- 标签匹配使用标准化后的完整名称，忽略英文大小写，不做模糊匹配、别名推测或 AI 猜测。
- 一个启用标签唯一匹配时，同时写入标签名称快照和真正的 `manualTagIds`。
- 标签不存在时保留“同步缺失标签”功能；同步完成后必须重新预检，匹配成功才允许导入。
- 不同分组存在同名启用标签时视为歧义，禁止导入并展示同名标签名单。
- 标签 ID 绑定由服务端根据当前标签目录再次验证，不能信任客户端直接提交的 ID。

### 4. 页面精简

- 删除“创建缺失员工”按钮、图标、处理函数和已暂停批量创建员工的提示。
- 保留“同步缺失来源”和“同步缺失标签”。
- 缺失负责人、重名负责人、缺失标签或重名标签存在时，导入按钮禁用，并在预检区域给出原因。
- 预检通过时明确显示“负责人可安全绑定”“标签可安全绑定”。

## 数据流

1. 用户选择 CRM 文件并执行预检。
2. 预检提取团队客户最终负责人和标签名称。
3. 预检分别生成唯一匹配、缺失和重名结果。
4. 任一负责人或标签存在缺失/重名时，页面禁止导入。
5. 用户补齐员工或同步标签后重新预检。
6. 正式导入时，服务端重新读取当前员工和标签目录，按名称唯一解析 ID。
7. 服务端发现目录已变化或匹配不唯一时拒绝整次导入，不写入部分客户。
8. 校验成功后，在同一导入事务中保存客户负责人 ID 和标签 ID。

## 错误处理

- 员工缺失：`以下负责人尚未创建员工：...`
- 员工重名：`以下负责人存在多个同名员工，无法确定归属：...`
- 标签缺失：`以下标签尚未同步：...`
- 标签重名：`以下标签在多个分组中重名，无法确定标签：...`
- 目录在预检后发生变化：服务端拒绝导入，提示重新预检。
- 校验失败时整批不写入，避免部分成功造成二次导入困难。

## 测试与验收

- 团队客户负责人缺失时，预检显示名单且导入不可执行。
- 团队客户负责人重名时，预检显示名单且导入不可执行。
- 公海客户不因原负责人缺失而被拦截。
- 唯一负责人导入后写入正确员工 ID；后来创建同名员工不改变归属。
- 标签名称唯一匹配时写入正确标签 ID，并在客户列表显示为正式标签。
- 缺失标签和跨分组同名标签均阻止导入。
- 标签拆分符和大小写兼容规则保持不变。
- 失效的批量创建员工入口完全移除，来源和标签同步入口保留。
- 全量自动化测试与生产构建通过。

## 明确不做

- 不按“第一个同名员工”自动绑定。
- 不在员工创建后自动认领历史客户。
- 不增加未匹配员工徽标或筛选器。
- 不提供批量创建员工。
- 不做标签模糊匹配、别名库或 AI 自动判断。
- 不自动清空或修改当前测试数据。
