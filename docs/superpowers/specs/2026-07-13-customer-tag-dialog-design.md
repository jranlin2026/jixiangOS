# 客户标签弹窗与线索标签下线设计

## 目标

客户管理保留唯一的人工标签能力，并将客户详情页的标签交互调整为分组弹窗；线索管理不再提供、读取或保留人工标签。

## 客户详情页

- 在客户姓名下方显示已选标签，使用标签目录中定义的颜色。
- 提供“+ 标签”入口；已选标签与入口保持同一行并自动换行。
- 点击入口后打开“设置标签”弹窗：左栏显示标签组和组内标签数量，右栏显示当前组的标签。
- 支持关键词搜索；搜索结果仍遵循标签颜色与选择状态。
- 目录中 `single` 组仅允许选择一个标签，`multiple` 组允许多选；沿用每条客户最多 20 个标签的现有规则。
- 点击“确定”才写入；“取消”不改变数据。
- 保存成功后立即更新客户资料，并在客户动态追加一条标签变更记录，包含变更前与变更后的标签名称。

## 数据与权限

- 继续使用现有客户标签目录、标签组、`manualTagIds` 与客户标签名称快照，不创建第二套标签模型。
- 客户标签目录只对客户范围生效；现有客户标签管理与权限继续有效。
- 线索的新建、编辑、详情和列表筛选中移除“预设标签”入口和展示。
- 清空所有现有线索记录中的 `manualTagIds` 与旧 `tags` 数据；不删除客户标签目录，也不修改客户标签。
- 清理同时覆盖本机 MySQL 中的线索记录和浏览器遗留线索缓存，避免测试数据回流。

## 实现边界

- 抽出可复用的客户标签弹窗组件，由客户详情页调用。
- 客户详情页不再把标签选择隐藏在“编辑资料”模式中；其他客户资料字段的编辑方式保持不变。
- 不改变客户导入的标签映射；导入产生的客户标签仍显示在新界面中。
- 不新增线索标签的替代功能。

## 验收标准

1. 客户详情页可通过“+ 标签”打开分组标签弹窗。
2. 单选/多选规则、搜索、取消与确定均符合标签目录配置。
3. 确定后客户页立即显示彩色标签，动态出现标签更新记录。
4. 线索页面、表单和详情不再出现预设标签；数据库与本地缓存中的线索标签数据为空。
5. 客户标签目录、已有客户标签和 EC CRM 导入标签不受影响。
6. 相关回归测试、生产构建及浏览器验证通过。
